# Neon Runner - 实际体验测试计划

## 1. 测试概述

**测试目的：** 对 Neon Runner 游戏进行实际运行体验测试
**测试日期：** 2026-03-28
**测试方法：** 自动化脚本模拟玩家操作 + 运行时状态监控

## 2. 测试环境

| 项目 | 详情 |
|------|------|
| Python 版本 | 3.13.8 |
| pygame 版本 | 2.6.1 |
| 游戏文件 | main.py, player.py, enemy.py, projectile.py, platform.py, particles.py, constants.py |
| 测试脚本 | test_experience_verification.py |

## 3. 测试场景

### 3.1 核心功能模块测试

| 测试项 | 测试内容 | 预期结果 |
|--------|----------|----------|
| 游戏启动 | 初始化 pygame，创建游戏窗口 | 窗口正常创建，标题为 "Neon Runner" |
| 游戏循环 | 运行游戏主循环 | 帧率稳定在 60 FPS |
| 玩家移动 | 使用方向键和AD键移动 | 玩家平滑移动，边界限制生效 |
| 跳跃功能 | 按空格键跳跃，包括二段跳 | 跳跃高度正确，二段跳可用 |
| 攻击功能 | 鼠标左键点击发射子弹 | 子弹向点击方向发射 |
| 敌人AI | 敌人自动追踪玩家 | 敌人持续向玩家移动 |
| 碰撞检测 | 子弹击中敌人，玩家接触敌人 | 敌人死亡，玩家受伤 |
| 粒子效果 | 攻击和死亡时粒子效果 | 粒子正确显示和衰减 |

### 3.2 用户界面交互测试

| 测试项 | 测试内容 | 预期结果 |
|--------|----------|----------|
| 生命值显示 | 左上角显示3个生命图标 | 图标正确显示，受伤后减少 |
| 退出功能 | 按 ESC 键退出游戏 | 游戏正常退出 |

### 3.3 性能表现测试

| 测试项 | 测试内容 | 预期结果 |
|--------|----------|----------|
| 帧率稳定性 | 监控游戏运行时的 FPS | 保持在 60 FPS |
| 内存使用 | 监控子弹和粒子积累 | 无内存泄漏 |
| 响应延迟 | 输入到响应的延迟 | 无明显延迟 |

## 4. 测试执行步骤

### 步骤 1：游戏初始化测试
- 启动游戏
- 验证窗口创建
- 验证所有组件初始化

### 步骤 2：玩家操作测试
- 模拟向左移动
- 模拟向右移动
- 模拟跳跃（二段跳）
- 模拟攻击

### 步骤 3：游戏逻辑测试
- 验证敌人追踪
- 验证碰撞检测
- 验证粒子效果

### 步骤 4：边界测试
- 验证玩家边界限制
- 验证子弹屏幕外清理
- 验证生命值下限

### 步骤 5：性能监控
- 记录 FPS
- 检查运行时错误

## 5. 缺陷分类

| 严重程度 | 分类 | 描述 |
|----------|------|------|
| P0 - 致命 | 游戏崩溃 | 导致游戏无法继续运行的严重错误 |
| P1 - 严重 | 功能缺失 | 核心功能无法正常工作 |
| P2 - 一般 | 功能异常 | 功能可用但表现不符合预期 |
| P3 - 轻微 | 体验问题 | 不影响功能但影响用户体验的问题 |
